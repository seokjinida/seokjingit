# Neon Pop Sweeper - Implementation Rules v1.2 (Current Build)

이 문서는 현재 구현 방향에서 AI 에이전트의 임의 해석을 막기 위한 고정 룰입니다.

## 1) 변경 금지 코어 범위

- 세션 시간: `70000ms`
- 맵 크기: `2000x2000`
- 인원 구성: 플레이어 1 + 봇 3
- 산출물: `index.html` 단일 파일
- 외부 라이브러리/이미지/폰트/네트워크 요청 금지

## 2) 현재 UX 고정 규칙

- 게임 진입 시 시작 패널을 먼저 보여주고 `Start Game`으로 시작
- 시작 직후 `2~3초` 내 참가자 합류 연출(카운트다운/토스트) 유지
- 이동 입력은 `Pointer + WASD/Arrow`를 모두 지원
- 랭킹은 실시간 갱신, 종료 시 결과 패널로 확정
- 우측 상단 랭킹 패널은 기존 기준 대비 20% 확대 상태 유지
- 좌측 하단 상태 패널(콤보/줌/점수)은 렌더링하지 않음
- 모바일/PC 공통으로 `16:9` 고정 게임 뷰(레터박스) 유지

## 3) 게임플레이 제약

- 공격/피격/체력/충돌 페널티 시스템 추가 금지
- PvP 상호작용(밀치기/방해) 추가 금지
- 성장/점수/콤보의 수치식은 `01_GAME_SPEC.md` 기준으로 유지
- AI 이름은 유저형 닉네임 규칙을 사용하고 기계형 이름(`Bot_1`, `AI_Player`) 사용 금지

## 4) 콤보/연출 규칙

- 콤보 체인 윈도우는 `0.6초` 고정
- 체인 텍스트는 화면 중앙 상단 UI 레이어에 표시
- 색상 단계와 크기 단계(15단위 + 10% 증분) 유지
- 수집 시 화면 흔들림은 현재 빌드에서 비활성(추가 금지)

## 5) 시간/카메라/월드 안정성

- 시간 계산은 `performance.now()` 사용
- 종료 판정은 `elapsedMs >= 70000`
- 카메라는 항상 플레이어 기준 + 월드 클램프 필수
- 어떤 줌/해상도에서도 맵 밖 미노출 보장

## 6) 성능 가드레일

- 동시 파티클 최대 `140`
- FPS가 45 미만으로 1초 지속 시 품질 하향 유지
  - 파티클 생성량 감소
  - `shadowBlur` 하향

## 7) 코드 품질

- 상수는 `CONFIG`에 모아서 관리
- 핵심 로직(입력/콜리전/카메라/UI/게임플로우) 섹션 분리 유지
- TODO/FIXME 남기지 않고 실행 가능한 상태 유지
- 시작 전 렌더 안정성 보장
  - 렌더 단계에서 빈 컬렉션 접근(`ranks[0].id` 등)으로 예외가 나지 않도록 방어
  - 또는 부트 시점에 최소 엔티티를 초기화해 렌더가 항상 안전하게 수행되도록 구성

## 8) 완료 조건

`docs/05_VALIDATION_CRITERIA.md`의 최신 기능 체크리스트/판정 기준을 통과하면 완료로 간주한다.

## 9) 에이전트 판단이 어려운 영역 가이드

- 아트 방향 우선순위:
  - 네온 스타일보다 가독성을 우선한다.
  - 텍스트/핵심 UI는 배경 대비를 충분히 확보한다.
  - 이펙트 강화로 플레이 정보(닷, 플레이어, 랭킹)가 가려지면 즉시 완화한다.
- 밸런싱 수치 조정 원칙:
  - 체감 난이도 이슈가 있어도 즉시 임의 변경하지 않는다.
  - 먼저 `docs/04_FUN_HYPOTHESES.md`와 `docs/05_VALIDATION_CRITERIA.md` 기준으로 원인을 분류하고, 문서를 선수정한 뒤 코드에 반영한다.
  - 한 번에 하나의 변수군(예: 이동속도 또는 스폰 밀도)만 조정해 영향도를 분리 검증한다.
- UX 판단 원칙:
  - 모바일/PC 공통으로 오조작 방지(터치 목표 크기, 버튼 간격)와 정보 위계(타이머/랭킹/체인)를 유지한다.
  - 새 UI 추가보다 기존 UI 가독성 개선을 우선한다.
- 검증 우선순위:
  - 기능 동작 여부 -> 안정성(에러/중단) -> 성능(FPS) -> 반복 실행 안정성 순으로 확인한다.
