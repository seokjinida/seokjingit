# Neon Pop Sweeper - Implementation Rules v2.6 (Current Build)

이 문서는 현재 구현 방향에서 AI 에이전트의 임의 해석을 막기 위한 고정 룰입니다.

## 1) 변경 금지 코어 범위

- 세션 시간: `60000ms`
- 맵 크기: `2000x2000`
- 인원 구성: 플레이어 1 + 봇 3
- 산출물: `NeonPopSweeper.html` 단일 파일
- 외부 라이브러리/이미지/폰트/네트워크 요청 금지

## 2) 현재 UX 고정 규칙

- 게임 진입 시 시작 패널을 먼저 보여주고 `게임 시작`으로 조인 씬 진입
- 시작 설명 문구는 `60초 동안 네온 닷을 모아 최고 점수를 노리세요.` 문자열로 고정한다.
- 시작 조작 안내 문구는 `이동: 드래그 조이스틱 / WASD / 방향키` 문자열로 고정한다.
- 시작 패널의 텍스트(제목/설명/안내)는 중앙 정렬을 유지
- `게임 시작` 버튼은 시작 카드 내부 중앙에 유지
- 조인 씬: 약 `5.0초` 진행 바 + 남은 시간 표시, AI 합류는 `0.4~4.4초` 랜덤, 미합류 봇은 비표시 · 합류 시 화면 밖 슬라이드 인
- 조인 씬/게임 시작 전에는 AI를 포함한 액터의 실제 이동 업데이트를 금지한다(합류 슬라이드 인 연출만 허용)
- 조인 슬라이드 인은 상태 업데이트만 하지 말고 화면에 실제 렌더되어야 한다.
  - 조인 단계 렌더 조건은 `joined`만 허용하지 않고 `joining || joined`를 만족하도록 구성한다.
- 조인 종료 후 `게임 시작!` HTML 오버레이(`~1.8초`) 다음 본편 타이머 시작
- 전체화면은 시작 패널의 별도 버튼으로만 토글 — 시작 카드 **외부 하단 중앙**에 배치하고 `게임 시작`으로 전체화면을 켜지 않음
- 시작 카드와 `전체화면` 버튼의 수직 간격은 `8px` 고정(`startOverlayShell gap`)으로 유지한다.
- 이동 입력은 `Pointer + WASD/Arrow`를 모두 지원
- 랭킹은 실시간 갱신, 종료 시 결과 패널로 확정
- 랭킹 패널은 **본편(`playing`)에서만 표시**하고 시작/조인/시작연출/결과 단계에서는 숨김 처리
- 랭킹 패널은 **게임 뷰 기준 좌측 상단**에 고정하고, 기존 기준 대비 20% 확대 상태를 유지
- 랭킹 패널 위치는 뷰포트 기준이 아니라 `gameView(view.x/view.y)` 기준으로 계산한다
- 화면비 변경 시 랭킹 패널 위치/크기는 `view.scale`에 맞춰 비례 조정한다
- 랭킹 패널은 스케일 적용 후에도 게임 뷰 경계를 벗어나지 않도록 좌표를 클램프한다
- 랭킹 패널 치수는 고정 상수로 하드코딩하지 말고, 실제 텍스트 측정(`measureText`) 결과 + 패딩으로 계산한다
- 텍스트/라인 수가 바뀌어도 랭킹 패널 테두리와 텍스트 블록이 항상 정합되도록 한다
- 랭킹 목록에서 플레이어(`YOU`) 항목은 노란색 강조 스타일을 강제한다
- 랭킹/HUD/결과 패널의 점수 표기는 정수(`floor`)로 고정, 소수점 노출 금지
- 좌측 하단 상태 패널(콤보/줌/점수)은 렌더링하지 않음
- 모바일/PC 공통으로 `16:9` 고정 게임 뷰(레터박스) 유지
- 액터 닉네임은 월드 상에서 머리 위 라벨로 표시(비합류 봇 제외)
- 1등 왕관은 닉네임 라벨과 겹치지 않도록 라벨 상단에 충분한 간격을 두고 렌더링한다

## 3) 게임플레이 제약

- 공격/피격/체력/충돌 페널티 시스템 추가 금지
- PvP 상호작용(밀치기/방해) 추가 금지
- 성장/점수/콤보의 수치식은 `01_GAME_SPEC.md` 기준으로 유지
- `chaser` / `hazard` 닷은 스펙에 정의된 동작(이동·페널티·분산·비수집 구간)을 벗어나 임의 확장하지 않음
- 문서에 정의되지 않은 닷 타입(예: `refund`)을 신규 추가하지 않는다.
  - 위험 닷 손실 환원은 `normal` 타입 기반의 보조 필드 처리로 구현하고, 타입 스키마는 5종(`normal/bonus/fever/chaser/hazard`)으로 고정한다.
- 붉은 위험 닷(`hazard`)의 동시 활성 개수는 최대 `3`을 초과하지 않도록 고정한다.
- AI는 붉은 위험 닷(`hazard`)을 인지/회피 대상으로 취급하지 않는다.
- AI 이동 로직에 위험 닷 기반 감지 반경/회피 확률/회피 벡터를 두지 않는다.
- AI 이름은 유저형 닉네임 규칙을 사용하고 기계형 이름(`Bot_1`, `AI_Player`) 사용 금지

## 4) 콤보/연출 규칙

- 콤보 체인 윈도우는 `0.6초` 고정
- `0.6초` 내 다음 닷을 획득하지 못하면 콤보를 즉시 `0`으로 초기화한다.
- 체인 텍스트는 화면 중앙 상단 UI 레이어에 표시
- 색상 단계와 크기 단계(15단위 + 10% 증분) 유지
- 콤보 텍스트 스케일 연출은 콤보 증가 시 **1회만** 재생한다.
- 1회 연출 최대 배율은 현재 단계 기본 스케일 대비 **2배**로 고정한다.
- 반복 펄스/무한 진동형 스케일 애니메이션은 금지한다.
- 콤보는 연출/점수 계산용으로만 사용하고, 캐릭터 스케일 계산과 분리한다.
- 캐릭터 스케일은 별도 성장값(`scaleLevel`)로 관리하며 최대값 `50`을 초과하지 않는다.
- 스케일 증가량은 닷 `comboGain`을 그대로 사용하지 않고 축소 비율(`×0.35`)을 적용한다.
- 수집 시 화면 흔들림은 현재 빌드에서 비활성(추가 금지)

## 5) 시간/카메라/월드 안정성

- 시간 계산은 `performance.now()` 사용
- 종료 판정은 `elapsedMs >= 60000`
- 카메라는 항상 플레이어 기준 + 월드 클램프 필수
- 어떤 줌/해상도에서도 맵 밖 미노출 보장
- 렌더 좌표 변환은 한 번만 적용
  - 게임 뷰 외곽 `translate/scale`을 적용한 뒤 월드 오브젝트를 그릴 때, 오브젝트 좌표 계산에서 `viewScale`/`gameView.x,y`를 중복 적용하지 않는다.
  - 플레이어, 닷, 파티클이 동일 좌표 변환 함수를 사용하도록 고정한다.
  - 화면비 변경 시 월드 최종 줌은 `effectiveZoom = cameraZoom * view.scale`를 사용하고, 플레이어/닷/배경이 동일하게 스케일되어야 한다.

## 6) 성능 가드레일

- 동시 파티클 최대 `140`
- FPS가 45 미만으로 1초 지속 시 품질 하향 유지
  - 파티클 생성량 감소
  - `shadowBlur` 하향

## 7) 코드 품질

- 상수는 `CONFIG`에 모아서 관리
- 핵심 로직(입력/콜리전/카메라/UI/게임플로우) 섹션 분리 유지
- TODO/FIXME 남기지 않고 실행 가능한 상태 유지
- 시작 전 렌더 안정성 보장
  - 렌더 단계에서 빈 컬렉션 접근(`ranks[0].id` 등)으로 예외가 나지 않도록 방어
  - 또는 부트 시점에 최소 엔티티를 초기화해 렌더가 항상 안전하게 수행되도록 구성
  - 초기 리사이즈/스케일 단계에서 접근하는 DOM 레퍼런스(`els.*`)는 모두 선언되어야 하며, `undefined.style` 예외가 발생하지 않도록 보장한다.
- 가시성 회귀 방지 체크
  - `게임 시작` 후 1초 이내 플레이어(YOU)가 화면 중앙 근처에 보이는지 수동 검증한다.
  - 플레이어/AI 닉네임 라벨이 월드에서 보이는지 수동 검증한다.

## 8) 완료 조건

`docs/05_VALIDATION_CRITERIA.md`의 최신 기능 체크리스트/판정 기준을 통과하면 완료로 간주한다.

## 9) 에이전트 판단이 어려운 영역 가이드

- 아트 방향 우선순위:
  - 네온 스타일보다 가독성을 우선한다.
  - 텍스트/핵심 UI는 배경 대비를 충분히 확보한다.
  - 이펙트 강화로 플레이 정보(닷, 플레이어, 랭킹)가 가려지면 즉시 완화한다.
- 밸런싱 수치 조정 원칙:
  - 체감 난이도 이슈가 있어도 즉시 임의 변경하지 않는다.
  - 먼저 `docs/04_FUN_HYPOTHESES.md`와 `docs/05_VALIDATION_CRITERIA.md` 기준으로 원인을 분류하고, 문서를 선수정한 뒤 코드에 반영한다.
  - 한 번에 하나의 변수군(예: 이동속도 또는 스폰 밀도)만 조정해 영향도를 분리 검증한다.
- UX 판단 원칙:
  - 모바일/PC 공통으로 오조작 방지(터치 목표 크기, 버튼 간격)와 정보 위계(타이머/랭킹/체인)를 유지한다.
  - 새 UI 추가보다 기존 UI 가독성 개선을 우선한다.
- UI 스케일 기준:
  - 게임 뷰(`16:9`) 축소 시 HUD/오버레이/버튼이 함께 비례 축소되어야 한다.
  - 기본 원칙은 비례 축소이며, 최근 확정값에 따라 시작/조인/결과 오버레이에 `min 0.78` 하한을 적용한다.
  - 시작 오버레이 셸은 `clamp(view.scale, 0.78, 1)`을 사용한다.
  - 조인/결과 카드는 `clamp(view.scale*1.02, 0.78, 1)`을 사용한다.
  - 시작 패널의 `전체화면` 버튼은 화면비 변경 후에도 항상 카드 바로 아래 위치를 유지한다.
  - UI 폰트는 문서 고정 토큰(`docs/01` 11-3)을 따른다. 대체 폰트/크기 체계로 전면 재작성하지 않는다.
- 검증 우선순위:
  - 기능 동작 여부 -> 안정성(에러/중단) -> 성능(FPS) -> 반복 실행 안정성 순으로 확인한다.

## 10) 콘텐츠별 세부 조건 고정 규칙 (원샷 빌드용)

- `docs/01_GAME_SPEC.md`의 `11. 콘텐츠별 세부 규격표`는 임의 축약/생략 없이 그대로 구현한다.
- 아래 항목은 **누락 시 즉시 미완료**로 판정한다.
  - 조인 씬 진행바(게이지) + 남은 시간 + 합류 피드
  - 시작 카드 외부 하단 중앙의 `전체화면` 버튼
  - hazard 경고 링(삼각 본체와 동시 렌더)
  - 수집 파티클의 spark + shard 2레이어
  - 닷/파티클 렌더 구간 `lighter` 합성 + 이후 `source-over` 복귀
  - 랭킹 패널의 텍스트 측정 기반 치수 계산(테두리-텍스트 정합)
  - AI 조인 슬라이드의 오프스크린 시작 보장(게임뷰 바깥 시작)
  - 조인 슬라이드 인의 가시 렌더 보장(`joining` 상태 노출)
  - 왕관-닉네임 최소 간격 보장(겹침 금지)
  - 문서 외 닷 타입 추가 금지(5종 타입 스키마 고정)
- UI/스케일 고정값:
  - 시작 오버레이 셸 스케일은 `clamp(view.scale, 0.78, 1)`을 유지한다.
  - 조인 카드 스케일은 `clamp(view.scale*1.02, 0.78, 1)`을 유지한다.
  - 결과 카드 스케일은 `clamp(view.scale*1.02, 0.78, 1)`을 유지한다.
  - 시작 패널은 카드 단독 스케일이 아니라 시작 오버레이 셸 단위로 동일 스케일 체인을 적용한다.
  - 콤보 기본 폰트는 `22.5 * view.scale`을 기준으로 렌더링한다.
  - `docs/01`의 파티클 티어 절대 수치(count/speed/life/spark/shard)를 임의 변경하지 않는다.
- 버튼/문구 고정값:
  - 결과 패널의 재시작 버튼 라벨은 `다시 시작`을 사용한다.
  - 콤보 텍스트는 `COMBO xN` 형식 외 다른 접미 표현을 허용하지 않는다.
- 수치 변경 규칙:
  - 콘텐츠별 세부 조건을 바꿔야 할 때는 코드보다 문서를 먼저 수정한다.
  - 한 번에 하나의 조건군만 변경하고, 변경 즉시 `docs/05_VALIDATION_CRITERIA.md` 항목 기준으로 재검증한다.

## 11) 드리프트 금지 규칙 (강제)

- 아래 항목은 원샷 생성 시 **절대 고정값**으로 취급한다.
