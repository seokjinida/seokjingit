# Neon Pop Sweeper - Prototype Validation Criteria v2.9

이 문서는 프로토타입 완성 여부를 판단하기 위한 기능 체크리스트와 판정 규칙을 정의합니다.

## 1) 기능 완료 체크리스트
- [ ] 시작 패널 -> `게임 시작` -> 조인 씬 -> `게임 시작!` 오버레이 -> 정상 라운드 진입
- [ ] `게임 시작` 버튼은 시작 카드 내부 중앙, `전체화면` 버튼은 시작 카드 외부 하단 중앙에 분리 배치됨
- [ ] 시작 카드와 `전체화면` 버튼의 수직 간격이 `8px` 고정으로 유지됨
- [ ] 시작 설명 문구가 `60초 동안 네온 닷을 모아 최고 점수를 노리세요.`로 고정되어 표시됨
- [ ] 시작 조작 안내 문구가 `이동: 드래그 조이스틱 / WASD / 방향키`로 고정되어 표시됨
- [ ] 시작 조작 안내 문구의 폰트 크기가 `16px` 기준으로 표시됨
- [ ] 시작 오버레이 셸 스케일이 `clamp(view.scale, 0.78, 1)`로 적용되어 최소 가독성이 유지됨
- [ ] 조인 씬: 약 5초 진행 바·남은 시간·합류 피드 표시, 미합류 AI는 화면에 보이지 않음, 합류 시 화면 밖에서 슬라이드 인
- [ ] 조인 슬라이드 인이 상태값만 갱신되는 것이 아니라, `joining` 구간에서 화면에 실제로 보이며 내부로 이동함
- [ ] 조인 씬/게임 시작 전에는 AI 포함 액터의 실제 이동이 발생하지 않음(합류 슬라이드 인만 표시)
- [ ] 조이스틱 + WASD/방향키 이동 모두 동작
- [ ] 네온 사각형 닷 수집/리스폰 정상 동작
- [ ] 닷 컬러별 획득 점수가 스펙과 일치함(예: 기본 청록 `+10`, 보너스 연두 `+25`, 피버 핑크 `+40`, 체이서 보라 `+55`, 위험 빨강 `0` + 5% 손실 규칙)
- [ ] 움직이는 보라색 닷(`chaser`)이 스폰/이동/소멸/수집 보상으로 정상 동작
- [ ] 붉은 삼각 위험 닷(`hazard`) 획득 시 획득 대상이 빨갛게 점멸하고 `floor(score×5%)`(보유 점수 한도) 손실이 적용됨
- [ ] 위험 닷으로 잃은 점수만큼 필드에 환원 닷이 분산 스폰되고, 발사 비행 구간에는 수집되지 않음
- [ ] 붉은 위험 닷(`hazard`)의 동시 활성 개수가 최대 `3`을 넘지 않음
- [ ] AI 이동 로직이 위험 닷(`hazard`) 인지/회피를 별도 수행하지 않음
- [ ] AI 닉네임이 유저형 이름 규칙을 따르고 기계형 이름(`Bot_1`, `AI_Player`)을 사용하지 않음
- [ ] 액터 닉네임 라벨(YOU + AI)이 월드에서 캐릭터 머리 위에 표시됨(비합류 봇 제외)
- [ ] 닷 획득 시 플레이어 크기가 증가(성장)하는 동작 확인
- [ ] 닷을 일정 시간 획득하지 않으면 플레이어 크기가 점진적으로 감소하는 동작 확인
- [ ] 플레이어 크기 성장값은 최대치(`scaleLevel = 50`)를 넘지 않음
- [ ] 콤보 0.6초 규칙 확인(0.6초 내 미획득 시 콤보 즉시 `0`, 체인 초기화)
- [ ] 콤보 색상/크기 단계 연출(15단위, 10% 증분) 확인
- [ ] 콤보 증가 시 텍스트 스케일 연출이 1회만 재생되고, 해당 단계 기본 크기 대비 최대 2배까지 확대 후 복귀함
- [ ] 닷 수집 시 닷 타입에 따른 파티클 연출이 구분되어 보임
- [ ] 위험 닷 환원 분산의 퍼짐 거리가 지나치게 짧지 않음
- [ ] 1등 왕관이 닉네임 라벨과 겹치지 않음
- [ ] 1등 왕관과 닉네임 라벨 사이 최소 간격이 `8px` 이상 유지됨
- [ ] 60초 정확 종료 및 결과 패널 표시
- [ ] 랭킹/HUD/결과 패널 점수가 정수로 표시됨(소수점 없음)
- [ ] 랭킹 UI가 `playing` 단계에서만 표시되고 시작/조인/시작연출/결과 단계에는 표시되지 않음
- [ ] 랭킹 패널이 게임 뷰(16:9) 좌측 상단 기준에 고정되고, 화면비 변경 시에도 게임 뷰 경계를 벗어나지 않음
- [ ] 랭킹 패널 폭/높이가 텍스트 측정 기반으로 계산되어, 테두리와 텍스트 블록이 불일치하지 않음
- [ ] 랭킹 목록에서 플레이어(`YOU`) 항목이 노란색으로 강조 표시됨
- [ ] UI 폰트가 문서 고정 토큰(`docs/01` 11-3: start title/desc/hint/button)과 일치함
- [ ] 닷 타입 스키마가 `normal/bonus/fever/chaser/hazard` 5종으로 유지되고, 문서 외 타입(`refund` 등)이 추가되지 않음
- [ ] Restart 후 초기화/재플레이 정상 동작
- [ ] 치명적 런타임 오류 없이(콘솔 에러/예외로 게임 중단 없음) 1회 이상 완주 가능
- [ ] 초기 로드/리사이즈 단계에서 `undefined.style` 계열 콘솔 에러가 발생하지 않음
- [ ] 게임 뷰 축소 시 HUD/오버레이/UI가 함께 비례 축소되고, 시작/조인/결과 오버레이는 `min 0.78` 하한 규칙을 만족함
- [ ] 화면비 변경 시 플레이어/닷/배경이 `effectiveZoom = cameraZoom * view.scale` 체인으로 함께 축소/확대됨

## 1-1) 콘텐츠별 세부 조건 체크리스트 (원샷 빌드용)

- [ ] `docs/01_GAME_SPEC.md`의 `11. 콘텐츠별 세부 규격표` 항목이 코드에 모두 반영됨
- [ ] 조인 씬에 제목/남은시간/진행바/합류 피드 4요소가 모두 표시됨
- [ ] 시작 패널에서 `전체화면` 버튼이 시작 카드 외부 하단 중앙에 별도 배치됨
- [ ] 시작 오버레이 셸 스케일이 `clamp(view.scale, 0.78, 1)` 범위에서 클램프됨
- [ ] AI 조인 슬라이드가 게임뷰 바깥에서 시작해 내부로 진입함(오프스크린 시작 보장)
- [ ] 조인 단계 렌더에서 `joining` 상태 AI가 실제로 표시되어 슬라이드 인 가시성이 확인됨
- [ ] hazard는 삼각형 본체와 경고 링(펄스)이 동시에 렌더됨
- [ ] 수집 파티클이 spark + shard 2레이어로 구분되어 보임
- [ ] 파티클 티어 수치(count/speed/life/spark/shard)가 `docs/01` 11-4 절대값과 일치함
- [ ] 닷/파티클 렌더 구간에서 `lighter` 합성이 적용되고, 액터/UI 렌더 전에 `source-over`로 복귀함
- [ ] 콤보 기본 폰트가 고정 px가 아닌 `view.scale` 연동으로 반응형 적용됨
- [ ] 조인/결과 카드 스케일이 `min 0.78, max 1` 범위에서 클램프됨
- [ ] 결과 패널 재시작 버튼 라벨이 한글 `다시 시작`으로 표기됨

## 1-2) 드리프트 방지 체크리스트 (강제)

- [ ] 시작 설명 문구가 정확히 `60초 동안 네온 닷을 모아 최고 점수를 노리세요.`로 표시됨
- [ ] 시작 조작 안내 문구가 정확히 `이동: 드래그 조이스틱 / WASD / 방향키`로 표시됨
- [ ] 시작 카드-전체화면 버튼 간격이 정확히 `8px`로 유지됨
- [ ] 시작 오버레이 셸 스케일이 `clamp(view.scale, 0.78, 1)` 고정값을 사용함
- [ ] 월드 렌더와 카메라 클램프 모두 `effectiveZoom = cameraZoom * view.scale` 체인을 사용함
- [ ] 파티클 티어 수치(normal/bonus/fever/chaser/hazard)가 `docs/01` 12-3 절대값과 전부 일치함
- [ ] 위 드리프트 항목 중 하나라도 FAIL이면 빌드 합격으로 판정하지 않음
- [ ] 랭킹 패널 치수가 `measureText` 기반 + 패딩 계산으로 구현되어 있음
- [ ] AI 조인 시작 좌표가 맵 내부 임의 오프셋이 아닌 게임뷰 외곽 기준으로 계산됨
- [ ] 조인 렌더 조건이 `joined` 단일 조건이 아니라 `joining || joined` 가시성 규칙을 만족함
- [ ] UI 폰트가 문서 고정 토큰과 일치하고 임의 폰트 리셋이 없음
- [ ] 1등 왕관-닉네임 라벨 최소 간격 `8px` 규칙을 만족함
- [ ] 코드 내 문서 외 닷 타입 문자열(`refund` 등)이 존재하지 않음

## 2) 판정 규칙

- 기능 체크리스트 전부 통과 시 "프로토타입 합격"으로 판정
- 미달 시, 수치 변경 전에 원인을 입력/카메라/UI 가독성/스폰 밀도 축으로 먼저 분류
- `1)` 또는 `1-1)` 또는 `1-2)`의 체크 항목 중 하나라도 FAIL이면 합격으로 판정하지 않는다.

## 3) KPI(정량 검증 지표)

- KPI-1. 완주 안정성: 연속 3회 플레이 시 3/3회 모두 60초 라운드를 오류 없이 완주
- KPI-2. 입력 신뢰성: 각 세션 시작 후 10초 이내 플레이어 이동 입력(포인터 또는 키보드)이 최소 1회 이상 정상 반영
- KPI-3. 재시작 안정성: Restart 3회 반복 후에도 시작 패널 -> 조인 -> 라운드 진입 플로우가 매회 정상 동작
- KPI-4. 점수 표시 일관성: 랭킹/HUD/결과 패널의 점수가 전 구간 정수(`floor`) 표기 유지
- KPI-5. 경쟁 가시성: 매 세션에서 플레이어+AI 총 4개 닉네임 라벨이 월드에 식별 가능하게 표시됨(비합류 구간 제외)

## 4) 변경 원칙

- 검증 기준 변경은 본 문서를 먼저 수정한다.
- 변경 시 `docs/01_GAME_SPEC.md`, `docs/02_IMPLEMENTATION_RULES.md`, `docs/03_AGENT_EXECUTION_PROMPT.md`를 동기화한다.
