# Neon Pop Sweeper (Dynamic Vision) - Game Spec v1.17 (Current Build)

이 문서는 게임 기획·수치·플로우에 대한 단일 기준(SoT) 스냅샷입니다. 루트 `NeonPopSweeper.html` 구현과 차이가 있으면 본 문서를 우선합니다.

## 1. 게임 개요

- Title: Neon Pop Sweeper (Dynamic Vision)
- Genre: Hyper-Casual Collection & Competition
- Session Time: 정확히 60초 (`60000ms`)
- Platform: HTML5 Canvas (Desktop + Mobile Web)
- 승리 조건: 제한 시간 종료 시 최고 점수
- 핵심 재미 가설: `docs/04_FUN_HYPOTHESES.md` 참조

## 2. 현재 Core Loop

1. 시작 패널에서 조작 안내 확인 후 `게임 시작` (선택: `전체화면` 토글 — 시작 버튼과 별도)
2. **조인 씬** 약 `5.0초`: 상단 게이지 + `게임 시작까지 n초`, AI는 `0.4~4.4초` 사이 랜덤 시각에 합류 로그 표시 · 미합류 봇은 화면에 그리지 않음 · 합류 시 화면 밖에서 슬라이드 인(`~0.65s` 이징)
3. 조인 UI 종료 후 **게임 시작 연출** HTML 오버레이 `게임 시작!` (`1.8초`, 약 `0.7초` 홀드 후 페이드)
4. 본편: 조작(드래그 조이스틱 또는 WASD/방향키)으로 이동
5. 네온 스퀘어(및 특수) 닷 수집 → 점수/콤보 상승(위험 닷은 예외)
6. 캐릭터 크기 성장 + 카메라 동적 줌 적용
7. AI 3인과 실시간 점수 랭킹 경쟁
8. 60초 종료 시 결과 패널 + 재시작

## 3. 월드/엔티티 스펙

### 맵
- 크기: `2000 x 2000`
- 카메라 클램프로 맵 밖 미노출 보장

### 플레이어/봇 공통
- 기본 이동속도: `220 units/s`
- 반지름: `(15 + scaleLevel * 1.0) * 2.0`
  - `scaleLevel`은 닷 획득 시 `comboGain × 0.35` 비율로 증가하고, 최대 `50`에서 고정된다.
  - 마지막 획득 후 `1.2초` 지나면 초당 `5`씩 감소한다.
- 콤보 최대: 연출용으로 누적되며 캐릭터 스케일 계산과 분리된다.
- 공격/충돌 페널티 없음(겹침 허용)

### 플레이어/봇 차등 밸런스
- 플레이어 속도 배수: `1.0`
- 봇 속도 배수: `0.96`
- 봇 조향: 목표 방향 즉시 전환(응답 패널티 없음)
- 플레이어 획득 배수: `1.82`
- 봇 획득 배수: `1.82` (플레이어와 동일)
- 획득 범위는 캐릭터 스케일과 함께 커지되, 캐릭터 외곽과 획득 경계 간격은 고정 오프셋 방식

### 닷(Dot)
- 형태: 네온 사각형
- 초기 생성 수: `220`
- 유지 범위: `220~280`
- 리스폰 검사 주기: `0.25s`
- 스폰 제한: 모든 캐릭터 기준 반경 `120` 이내 금지
- 붉은 위험 닷(`hazard`) 동시 활성 최대: `3`

| Type | Spawn Ratio | Base Score | Combo Gain | Extra |
|---|---:|---:|---:|---|
| normal | 82% | +10 | +1 | - |
| bonus | 12% | +25 | +2 | - |
| fever | 3% | +40 | +4 | 피버 2.5초 |
| chaser | 2% | +55 | +5 | 보라색 이동 닷, 속도 `78~104` units/s, 벽 반사, 수명 `6~8초` 후 소멸 |
| hazard | 1% | 0 | 0 | 붉은 삼각 위험 닷(펄스·경고 링), 획득 시 `floor(score × 5%)` 점수 차감(보유 점수 한도), 동일 분량을 일반 닷으로 주변 분산 · 발사 구간(`~1.15s`) 비획득 |

## 4. 점수/콤보/연출 규칙

### 점수
- 수집 시: `score += baseScore + combo * 2`
- 예외: `hazard` 수집 시 점수를 획득하지 않고, `actualLost = min(score, floor(score * 0.05))` 만큼 차감한다. (점수 `0`이면 손실·분산 없음)

### 콤보
- 수집 시: `combo = min(999, combo + comboGain)`
- 체인 연출 유지시간: `0.6초`
  - 0.6초 내 연속 수집 시 콤보/체인 연출 유지
  - 0.6초 내 다음 닷을 획득하지 못하면 콤보는 즉시 `0`으로 초기화
  - 0.6초 초과 시 체인 수 즉시 초기화
- 콤보는 UI/점수 연출용 지표이며 캐릭터 크기 계산에는 사용하지 않는다.

### 콤보 텍스트(중앙 상단 UI)
- 위치: 화면 중앙 상단 (`y=170`)
- 컬러 단계(15단위):
  - `1~15` 흰색
  - `16~30` 파랑
  - `31~45` 노랑
  - `46~60` 빨강
  - `61+` 보라
- 색 단계가 올라갈수록 크기 10%씩 증가
- 콤보 증가 시 스케일 연출은 **1회만 재생**한다.
- 스케일 연출 크기는 현재 단계 기본 크기 대비 **최대 2배**까지 확대 후 기본 크기로 복귀한다.
- 상시 진동/반복 펄스 애니메이션은 사용하지 않는다.

### 피버(2.5초)
- 수집 반경 `+35%`
- 이동속도 `+10%`
- 파티클 강화

## 5. 카메라 스펙 (현재 튜닝값)

- `targetZoomRaw = (1 / (1 + scaleLevel * 0.007)) * 1.3`
- 보간: `currentZoom = lerp(currentZoom, targetZoom, 0.1)`
- 줌 범위: `0.2 ~ 3.0`
- 시작 줌: `3.0` (초기 카메라를 더 가깝게)
- 화면 비율: `16:9` 고정 게임 뷰(레터박스 적용)
- 화면 축소 시 전체 뷰 스케일 동기화:
  - 기준 폭 `1600` 대비 비율로 월드/UI/조이스틱이 함께 축소
  - 작은 화면일수록 플레이어/닷/UI가 상대적으로 작아지고, 화면에 보이는 월드 범위가 넓어짐
- 월드 클램프:
  - `halfW = gameView.width / (2 * zoom)`
  - `halfH = gameView.height / (2 * zoom)`
  - `camX = clamp(player.x, halfW, MAP_W - halfW)`
  - `camY = clamp(player.y, halfH, MAP_H - halfH)`
- 좌표계 규칙(재발 방지):
  - 월드 오브젝트 렌더링은 **단일 좌표계**를 기준으로 수행한다.
  - `renderFrame`에서 `translate/scale`로 게임 뷰 변환을 이미 적용했다면, `worldToScreen`류 함수에서 `gameView.x/y`나 `viewScale`을 다시 곱하지 않는다.
  - 플레이어/닷/파티클은 동일 변환 체인(월드 -> 게임뷰 내부 좌표)을 공유한다.

## 6. 입력 스펙

- Pointer: `pointerdown`, `pointermove`, `pointerup`, `pointercancel`
- Keyboard: `WASD` + `Arrow Keys`
- 우선순위: 키보드 입력이 있으면 키보드 벡터 사용, 없으면 조이스틱 사용
- 조이스틱 최대 반경: `80px`

## 7. AI 봇 규칙

- 타깃 재선정 주기: `0.4초`
- AI는 위험 닷(`hazard`)을 별도 인지 대상으로 취급하지 않는다.
- 위험 닷(`hazard`) 기반 회피/가중치/우선순위 보정 로직을 사용하지 않는다.
- 평가식:
  - `valueScore = dotBaseScore * 1.0`
  - `distScore = -distance * 0.08`
  - `catchupBonus = (botRank >= 3) ? 8 : 0`
  - `total = valueScore + distScore + catchupBonus`
- 선택된 목표 방향으로 즉시 전환 이동
- 닉네임 규칙:
  - `Bot_1`, `AI_Player` 같은 기계형 이름 금지
  - 사람 닉네임처럼 보이는 4~10자 이름 사용(영문/숫자/언더스코어 소량 허용)
  - 예시: `NeoFox`, `MintBug`, `RinRin`, `JAY_92`

## 8. 렌더링/아트

- 배경: 페이지/레터박스 `#050512`, 캔버스 월드 클리어 `#0b0b1a`
- 네온 연출: `globalCompositeOperation = "lighter"`, `shadowBlur 16` (저사양 10)
- 파티클: 최대 `140`, FPS 저하 시 생성량·블러 완화
- **수집 파티클**: 닷 타입별 티어(`normal` / `bonus` / `fever` / `chaser` / `hazard`) — 개수·초기 속도·수명·크기·색상 상이, 저사양 시 개수 `×0.7` 추가 적용, 대형 샤드 레이어 병행
- 파티클 이동: 매 프레임 속도 `×0.93` 감쇠(구현 기준)
- 1등 캐릭터 상단 금색 왕관
- 액터 닉네임은 월드 상에서 각 캐릭터 머리 위에 항상 표시된다(조인 전 비합류 봇 제외).
- 획득 시 화면 흔들림: 제거됨(현재 미사용)
- 위험 닷 획득 시 액터 `~0.45s` 붉은 플래시

## 9. UI/플로우

### 시작 전
- 시작 안내 패널 표시
- 시작 설명 문구는 아래 고정 문자열을 사용한다: `60초 동안 네온 닷을 모아 최고 점수를 노리세요.`
- 시작 조작 안내 문구는 아래 고정 문자열을 사용한다: `이동: 드래그 조이스틱 / WASD / 방향키`
- **시작 패널 정렬**: 제목/설명/안내 문구/기본 액션은 중앙 정렬을 유지한다.
- **시작 패널 스케일**: 시작 오버레이 셸은 `clamp(view.scale, 0.78, 1)`을 적용해 최소 가독성을 유지한다.
- **전체화면**: 시작 카드 내부 기본 액션(게임 시작)과 분리된 **카드 외부 하단 중앙** 별도 버튼으로 토글(라벨: 전체화면 ↔ 창화면). `게임 시작` 버튼은 전체화면을 호출하지 않음. iOS 등 미지원 시 안내 문구 표시
- 조작법/라운드 안내 후 `게임 시작`으로 조인 씬 진입
- **조인 씬** (`JOIN_TOTAL_SEC = 5.0`)
  - 제목: "참가자 입장 중...", 하단 진행 바, `게임 시작까지 n초`
  - AI 합류 시각: 각 봇 `0.4 ~ min(4.4, JOIN_TOTAL_SEC-0.2)` 초 사이 랜덤
  - 합류 피드 로그 + 미합류 봇은 렌더 생략 · 합류 시 화면 밖에서 슬라이드 인
- 조인 종료 후 **게임 시작** 전용 패널(`GAME_START_FX_SEC = 1.8`): "게임 시작!" 텍스트 페이드
- 시작/결과/조인/게임시작 오버레이 UI는 게임 뷰 스케일과 동기화(작은 화면에서 함께 축소)
- 시작 전(플레이 미진입) 상태에서도 렌더 루프가 에러 없이 동작해야 함
  - 최소 엔티티 초기화 또는 빈 배열 방어 로직 중 하나를 반드시 포함

### 플레이 중
- 상단 중앙: 남은 시간(소수점 1자리)
- 랭킹 보드(1~4위, 기존 대비 20% 확대)
  - **게임 뷰(16:9) 기준 좌측 상단**에 고정 배치한다. (뷰포트 절대좌표 고정 금지)
  - 화면비 변경 시 `view.scale`에 따라 위치/크기가 함께 비례 축소되어야 한다.
  - 랭킹 패널은 게임 뷰 영역 밖으로 벗어나지 않도록 게임 뷰 경계 기준 클램프를 적용한다.
  - 랭킹 항목에서 플레이어(`YOU`)는 노란색 강조로 표시한다.
  - 점수 표기는 정수(`floor`)로 표시한다. 소수점 노출 금지
- 좌측 하단 상태 패널(콤보/줌/점수): 미표시

### 종료
- 조건: `elapsedMs >= 60000`
- 결과 패널: 최종 순위/점수, 플레이어 강조, `다시 시작`
- 동점: 먼저 해당 점수 도달한 참가자 우선

## 12. 드리프트 방지 고정값 (회귀 금지)

아래 항목은 원샷 생성 과정에서 자주 드리프트가 발생했던 값입니다.  
본 항목은 `11`장의 요약이 아니라 **회귀 금지용 우선 고정값**이며, 값/문구를 의역/치환하지 않습니다.

### 12-1) 고정 문구 (문자열 완전 일치)

- 시작 설명 문구: `60초 동안 네온 닷을 모아 최고 점수를 노리세요.`
- 시작 조작 안내 문구: `이동: 드래그 조이스틱 / WASD / 방향키`

### 12-2) 고정 레이아웃/스케일

- 시작 카드-전체화면 버튼 간격: `8px` (`startOverlayShell gap`)
- 시작 패널 스케일: `clamp(view.scale, 0.78, 1)`
- 월드 최종 줌: `effectiveZoom = cameraZoom * view.scale`
  - 카메라 클램프(halfW/halfH) 계산과 월드 렌더 스케일에 동일 적용

### 12-3) 고정 파티클 절대값

- `normal`: count `12~20`, speed `520~1000`, life `0.26~0.42`, spark `2.2~4.4`, shard `5.3~9.3`
- `bonus`: count `20~30`, speed `720~1360`, life `0.32~0.52`, spark `2.8~5.2`, shard `6.2~10.9`
- `fever`: count `28~40`, speed `880~1680`, life `0.38~0.62`, spark `3.1~5.8`, shard `7.0~12.2`
- `chaser`: count `32~44`, speed `1040~1840`, life `0.42~0.70`, spark `3.4~6.2`, shard `7.4~13.0`
- `hazard`: count `16~24`, speed `640~1200`, life `0.30~0.50`, spark `2.6~4.8`, shard `5.8~10.1`

### 12-4) 실패 판정 규칙

- 위 12-1/12-2/12-3 항목 중 하나라도 다르면 구현 완료로 간주하지 않는다.

### 12-5) UI/조인/랭킹/왕관 회귀 금지값

- 랭킹 패널은 고정 폭/고정 높이 상수만으로 만들지 않는다.
  - `measureText` 기반 텍스트 블록 크기 + 패딩으로 패널 치수를 계산한다.
- AI 합류 슬라이드 시작 좌표는 맵 좌표 임의 오프셋이 아니라, **현재 게임뷰 바깥**에서 계산한다.
  - 시작 프레임에서 최소 한 축(x 또는 y)이 게임뷰 경계 밖이어야 한다.
- UI 폰트 토큰(`start title/desc/hint/button`)은 11-3 고정값을 유지한다.
  - 문서 없는 폰트 패밀리/크기 리셋 금지.
- 1등 왕관은 닉네임 라벨과 분리 렌더한다.
  - 왕관/닉네임의 y 오프셋을 독립 계산하고 최소 간격 `8px`을 보장한다.
